1 Lisp基本概念

1.1 列表与原子

1.1.1 什么是列表？（列表由什么组成？）

列表的精彩之处在于，它不仅仅是Lisp中的一种数据结构，它也是Lisp语言的构成部分，Lisp语言中的所有语句都是一个列表。

列表的组成:

  1. 括号和由空格分隔的原子
  2. 括号的其他列表
  3. 仅仅括号 (空列表)

列表原子间的空格数量有限制吗，除了空格外，还可以使用什么分割原子？（所有的空白字 符）.

1.1.2 对列表进行排版

列表的排版缩进命令？ (M-C-\)

1.1.3 空列表

空列表既是原子也是列表

1.1.4 S-表达式

原子与列表的组成也叫表达式(symbolic expression | s-expression)。

1.1.5 原子

在Lisp中,原子代表一种不可再分的数据结构,也就是Lisp组成的最小单元.

涉及列表操作的原子都是不可分的。

1.1.5.1 原子的种类。
  • 数字
  • 符号
  • 字符串
1.1.5.2 求值原子

当对原子进行求值操作(按下 C-x C-e )时,emacs会将原子看成一个变量,试图从中获取一个值.

1.1.5.3 符号

符号是原子一种,它有两种特殊的用途:存放数值;存放函数.函数和数值可以同时放入一个符 号中,它们互不影响,emacs会根据表达式来判断其表示的是变量还是函数.

1.1.6 运行列表

任何的一个列表都是一个准备运行的程序,解释器会将列表的第一个符号解释为命令,执行它,若 执行过程中出现错误,那么会返回一些错误信息.

有一种特殊情况,当列表像下面这样,它并不运行,而是仅仅返回列表本身. 

'(hello I am a lisper)

它等价于调用命令 quote 

(quote hello I am a lisper)

在圆括号后面按下 C-x C-e emacs就会运行这个命令.

1.2 注释

注释以 ; 开头,一直到行尾.

1.2.1 单个分号构成的注释

适合放在代码后面,并且能够更好在右边进行右对齐.主要目的是说明位于改行的代码的具体 含义. 

(setq base-version-list                ; There was a base
      (assoc (substring fn 0 start-vn) ; version to which
             file-version-assoc-list)) ; this looks like
                                       ; a subversion.

1.2.2 两个分号的注释

这种注释与下一行代码具有相同的缩进等级,可以更好的描述下一行代码的含义或者程序允 许到该位置的状态. 

(prog1 (setq auto-fill-function
             ...
             ...
    ;; Update mode line.
    (force-mode-line-update)))

还有一种使用方式:放在函数头,这是为了说明一个没有文档的函数的工作状态. 

;; This Lisp code is run in Emacs when it is to operate as
;; a server for other processes.

1.2.3 三个分号的注释

Comments that start with three semicolons, ‘;;;’, should start at the left
margin. We use them for comments which should be considered a heading by
Outline minor mode. By default, comments starting with at least three
semicolons (followed by a single space and a non-whitespace character) are
considered headings, comments starting with two or fewer are not. Historically,
triple-semicolon comments have also been used for commenting out lines within a
function, but this use is discouraged.

以三个分号’;;;’开头的注释应从左边开始。我们将其用于应被视为大纲小模式标题的注释。 默认情况下，至少以三个分号开头的注释是(后面是一个空格和一个非空格字符)被认为是标 题，而以两个或更少的字符开头的注释则不被认为是标题。历史上，三分号注释也曾用于注 释函数中的行，但不鼓励使用。

1.2.4 四个分号的注释

以四个分号 ;;;; 开头的注释应向左边缘对齐，用于程序主要部分的标题。

2 Lisp数据结构

2.1 Lisp对象是什么?

Lisp对象就是被Lisp程序操作和使用的数据部分.

2.2 Lisp对象只能是一个类型吗?

Every object belongs to at least one type. Objects of the same type have similar structures and may usually be used in the same contexts. Types can overlap, and objects can belong to two or more types. Consequently, we can ask whether an object belongs to a particular type, but not for the type of an object.

Lisp对象只能是一个基本类型.

2.3 基本类型(primitive types)

These, from which all other types are constructed, are called /primitive types/.
这些，所有其他类型都是由这些类型构造出来的，被称为基元类型。

2.3.1 Lisp有哪些基本类型?

integer, float, cons, symbol, string, vector, hash-table, subr, byte-code function, record, Editor type

2.3.2 基本类型如何检查对象是否属于它?

每个基本类型都有一个对应的Lisp函数，用于检查一个对象是否是该类型的成员。

2.3.3 Lisp如何分辨一个对象的类型?

创建一个Lisp对象时不像其他语言一样需要显示定义它的类型,因此Lisp通过对象的初始值 来判断它是什么类型.

2.4 打印表达式和读入语法

每个类型都具有其表现形式(打印表达式)和被识别的形式(读入语法).

2.4.1 什么是打印表达式?

打印表达式是Lisp打印程序(例如 print1)输出对象的一个呈现形式.

每个对象都具备唯一的打印表达式.

2.4.2 什么是读入语法?

读入语法是Lisp读入程序读入一个对象时能够识别出其类型的语法格式.

一个对象(所属的类型)不仅仅有一个读入语法.

2.4.3 哪些类型具有打印表达式,哪些具有读入语法?

since it does not make sense to enter objects of these types as constants in a
Lisp program.
                                                make sense 有意义的
因为在Lisp程序中把这些类型的对象作为常量输入是没有意义的。

每个类型都具备打印表达式,而只有那些需要作为常量在Lisp程序中被使用的类型才具有读 入语法.

那些没有读入语法的类型的打印表达式都是以一种叫做散列符号(hash notation)的形式 被打印出来.例如: 

(current-buffer)
⇒ #<buffer objects.texi>

当试图读入散列符号时(以 #< 开头的序列),Lisp读入程序会被触发 invalid-read-syntax 的错误.

2.4.4 一个Lisp表达式

In other languages, an expression is text; it has no other form. In Lisp, an
expression is primarily a Lisp object and only secondarily the text that is the
object’s read syntax.

在其他语言中，表达式就是文本；它没有其他形式。在Lisp中，一个表达式主要是一个Lisp 对象，其次才是对象的读语法的文本。

2.4.5 对一个Lisp表达式求值

当我们对一个Lisp进行求值时,Lisp读入程序会从文本中读入它,然后产生一个Lisp对象,当 没有任何错误被发现时,Lisp返回这个对象的值.
